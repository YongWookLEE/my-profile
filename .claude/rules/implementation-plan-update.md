# implementation-plan.md 업데이트 규칙

## 언제 업데이트하는가

다음 작업을 수행한 후 반드시 `specs/implementation-plan.md`를 업데이트한다:

- 새 컴포넌트 또는 파일 생성
- 기존 컴포넌트/파일의 구조적 변경 (리팩토링, 기능 추가, 아키텍처 변경)
- TypeScript 인터페이스 변경 (필드 추가/제거/수정)
- 프로젝트 구조 변경 (폴더 추가, 파일 이동)
- 의존성 추가 또는 제거

## 업데이트 항목

변경 내용에 따라 해당 섹션을 수정한다:

1. **프로젝트 구조** — 신규 파일/폴더는 트리에 추가, 삭제된 항목은 제거
2. **TypeScript 인터페이스** — 변경된 인터페이스 코드블록 반영
3. **구현 체크리스트** — 완료된 항목은 `[x]`로 표시, 신규 항목은 적절한 섹션에 추가
4. **버전 히스토리** — 새 항목을 테이블에 추가 (형식: `| vX.Y | YYYY-MM-DD | 변경 요약 |`)

## 버전 규칙

- **minor (x.Y)**: 기능 추가, 아키텍처 변경, 컴포넌트 추가
- **patch (x.y.Z)**: 버그 수정, 스타일 조정, 텍스트 변경
- 버전은 기존 히스토리의 마지막 버전에서 증가

## 주의사항

- 업데이트는 코드 변경 완료 후, 커밋 전에 수행한다
- 변경 요약은 한국어로 작성한다
- 섹션별 상세 계획(Hero, Projects, Skills, Contact)은 기능이 크게 바뀔 때만 수정한다
