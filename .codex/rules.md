# Codex 작업 규칙

## 적용 범위

- 이 문서는 Codex가 이 저장소에서 작업할 때 따르는 전용 규칙이다.
- Claude 전용 규칙 파일(`./.claude/rules/*.md`)은 Codex 작업 기준으로 읽지 않는다.

## 코드 스타일

- 변수명은 `camelCase`를 사용한다.
- 함수명은 동사로 시작한다.
- 코드 주석과 작업 문서는 한국어로 작성한다.
- 변수명과 함수명은 영어를 사용한다.

## Git 규칙

- 커밋 메시지는 한국어로 작성한다.
- PR 제목과 본문은 한국어로 작성한다.
- 브랜치명은 기본적으로 `feature/기능명`, `hotfix/기능명` 형식을 사용한다.
- Codex 실행 환경에서 브랜치 prefix 제약이 있을 때는 `codex/feature/기능명`, `codex/hotfix/기능명` 형식을 허용한다.
- `main` 브랜치에는 직접 push 하지 않는다.
- 커밋 전 반드시 `npm run build`를 실행해 빌드 오류가 없는지 확인한다.

## 문서 업데이트 규칙

- 다음 작업 후에는 `specs/implementation-plan.md`를 업데이트한다.
- 새 컴포넌트 또는 파일 생성
- 기존 컴포넌트/파일의 구조적 변경
- TypeScript 인터페이스 변경
- 프로젝트 구조 변경
- 의존성 추가 또는 제거

## implementation-plan 업데이트 항목

- 프로젝트 구조: 신규 파일/폴더 추가, 삭제 항목 제거
- TypeScript 인터페이스: 변경된 인터페이스 반영
- 구현 체크리스트: 완료 항목 표시, 신규 항목 추가
- 버전 히스토리: `| 버전 | 날짜 | 변경사항 |` 형식으로 추가

## 버전 규칙

- `minor (x.Y)`: 기능 추가, 아키텍처 변경, 컴포넌트 추가
- `patch (x.y.Z)`: 버그 수정, 스타일 조정, 텍스트 변경, 문서 정리
- 버전은 기존 히스토리의 마지막 버전에서 증가시킨다.
